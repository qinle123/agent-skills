This is a subcommand alias for openspec-workflow.

## Instructions

The user wants to run the `continue` workflow for an OpenSpec change.

1. Load the `openspec-workflow` skill with `skill_view('openspec-workflow', file_path='references/opsx-continue.md')`
2. Follow its instructions step by step
3. The user's argument after `/openspec-continue` is the change name
