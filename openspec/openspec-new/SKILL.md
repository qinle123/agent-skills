This is a subcommand alias for openspec-workflow.

## Instructions

The user wants to run the `new` workflow to create an OpenSpec change step by step.

1. Load the `openspec-workflow` skill with `skill_view('openspec-workflow', file_path='references/opsx-new.md')`
2. Follow its instructions step by step
3. The user's argument after `/openspec-new` is the change name
