# 2026-07-18 Chapter Selfcheck Update Design

针对 `chapter-selfcheck` 技能进行迭代更新，优化人物细节与历史剧情的关联审计流程，并引入与 VS Code 一致的字数统计技术规范。

## 1. 目标与背景 (Objectives)

本次更新包含两个核心改进点：
1. **角色历史人设与场景回溯（高效审计原则）**：
   - 之前仅从静态人设文档比对年龄/辈分，容易忽略角色在历史章节/剧情中已发生的情境（例如历史出场场景、心理变化与独特经历）。
   - 为避免盲目全文检索导致效率低下，引入基于特定文件的定向检索逻辑（如检索控制卡 `control-cards/`、`03-cast-bible.md`、`04-relationship-map.md`、`08-dynamic-state.md`），以还原该角色已有的场景经历与定位，确保目标章节中的言行与其历史情境不冲突。
2. **字数统计技术规范（同 VS Code 逻辑）**：
   - 规定在审查涉及字数指标的风格指南或其他规范时，必须使用与 VS Code 相同的计数算法（中文字符单独计，英文/西文按单词计，标点按 VS Code 默认处理），除非项目有特定的例外规定。

## 2. 修改范围 (Scope of Changes)

1. **更新技能文件**：
   - 修改 `skills/chapter-selfcheck/SKILL.md` 中的 `步骤 1：人物、辈分称呼与年龄逻辑专项审计`，增加定向的高效角色信息与历史场景检索，并优化核对细则。
   - 在 `skills/chapter-selfcheck/SKILL.md` 文档末尾新增 `## 4. 字数统计技术规范 (Word Count Technical Standards)` 章节。

## 3. 详细设计 (Detailed Design)

### 3.1 `skills/chapter-selfcheck/SKILL.md` 更新内容

#### 3.1.1 步骤 1 (人物、辈分称呼与年龄逻辑专项审计) 更新
```markdown
### 步骤 1：人物、辈分称呼与年龄逻辑专项审计 (Character, Address & Age Logic Audit)
* **任务**：提取本章中出现或被提及的所有人物。
* **角色档案与历史背景精准定位（高效查找原则）**：
  - **禁止盲目全文检索**：针对提取出的每个角色，禁止对整个项目进行低效的全文检索。
  - **定向文件检索**：使用文件名或关键字定位，精准读取 `03-cast-bible.md`（角色设定）、`04-relationship-map.md`（角色关系）、`08-dynamic-state.md`（动态状态），以及在 `control-cards/` 目录中搜索该角色曾出现过的历史控制卡文件。
  - **历史剧情与人设还原**：确认该角色在本篇小说中已发生的所有出场场景、经历的剧情发展、以及核心人设定位。
* **核对**：
  - 从上述定向载入的设定与控制卡中查询这些人物的年龄、辈分、亲属/师门关系。
  - **称呼核对**：仔细研读角色之间的对话与旁白。核对彼此的称呼是否符合辈分和关系（例如：表哥/堂弟、师叔/师侄、叔侄、祖孙等称呼是否混淆）。
  - **人设与剧情一致性核对**：核对目标章节中该角色的言行举止、心理描写、语气语言，是否与其经历的剧情历史和既有人设相冲突，防止出现人物逻辑前后错乱。
  - **年龄与称呼逻辑核对**：重点检查“设定的年龄大小”与“口头称呼”是否逻辑自洽。例如，不能出现“年龄设定为15岁的小辈直呼50岁的长辈为贤弟”，或者“设定为妹妹的角色在对话里突然叫哥哥为弟弟”等称呼错位。
```

#### 3.1.2 文档末尾新增 `## 4. 字数统计技术规范`
```markdown
## 4. 字数统计技术规范 (Word Count Technical Standards)

在对照风格指南或项目其他规定了字数限制（如单段字数、单章字数、大纲字数等）的地方，必须严格遵循技术精确计数原则，采用与 VS Code 相同的统计逻辑：
1. **中文字符（CJK 统一汉字）**：每个中文字符（汉字）计为 1 个字。
2. **西文/英文单词**：按空格及词汇边界分隔进行计数，即一个完整的英文/西文单词（如 "VS Code"）计为 2 个字，而非按字母计数。
3. **标点符号与特殊字符**：遵循 VS Code 默认计数行为（通常标点符号不计入英文单词，但在中文字数统计时，全角标点符号会作为字符计入字数），除非项目（如 `09-style-guide.md`）中有特别针对字数排除标点或特殊符号的规定。
4. **统一性**：所有涉及字数限制的审计，均以此计数标准为准。
```
